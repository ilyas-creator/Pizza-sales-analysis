 Analyse des ventes de pizzas - Projet BI complet
Ce projet explore les données de vente d'une pizzeria pour en tirer des insights concrets. Il retrace tout le parcours des données, du nettoyage initial jusqu'à la création d'un tableau de bord interactif.

 L'objectif du projet
L'idée était de répondre aux questions clés du business (les KPIs) et de visualiser les tendances pour aider la direction à optimiser ses opérations et sa stratégie marketing.

 Méthodologie et outils utilisés
Nettoyage des données : Microsoft Excel (pour gérer les valeurs manquantes, uniformiser les dates et formater les données).

Analyse : SQL Server (avec des requêtes complexes pour extraire les KPIs et identifier les tendances).

Visualisation : Microsoft Excel (via des Tableaux et Graphiques Croisés Dynamiques) pour construire un tableau de bord interactif.

 Les questions business auxquelles on a répondu (KPIs)
L'analyse a été structurée pour répondre aux points suivants :

Chiffre d'affaires total : La somme du prix de toutes les commandes de pizzas.

Valeur moyenne d'une commande : Le montant moyen dépensé par commande.

Nombre total de pizzas vendues : La somme des quantités de toutes les pizzas vendues.

Nombre total de commandes : Le nombre distinct de commandes passées.

Nombre moyen de pizzas par commande : La quantité moyenne de pizzas vendues par commande.

Ce qu'on a découvert grâce au tableau de bord
Les jours les plus chargés : Les commandes atteignent leur pic le week-end (les vendredi et samedi soir).

Les heures de pointe : Le plus grand nombre de commandes a lieu entre 12h et 13h, puis après 18h.

Les best-sellers : Les pizzas "Classic Deluxe" et "Chicken" sont celles qui génèrent le plus de revenus.

Les moins performantes : La pizza "Brie Carre" arrive en dernière position, à la fois en nombre de commandes et en chiffre d'affaires.

 Organisation du projet
Data/ : Les jeux de données bruts et nettoyés.

SQL_Queries/ : Les scripts SQL utilisés pour extraire les KPIs.

Dashboard/ : Le classeur Excel final contenant les Tableaux Croisés Dynamiques.

Screenshots/ : Des captures d'écran du tableau de bord et des résultats SQL.
